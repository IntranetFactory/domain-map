# Plan: domain_kind taxonomy

Status: ready to execute. No schema or data changes yet. Review gaps resolved: A1/Rule #8 bundle carve-out (§1, §3, §4), HVAC industry = NAICS 238220 (§3), skill-repoint rationale corrected to consistency-not-F2 (§3), `cost_band` estimated (REAL-ESTATE-AGENT `$`, HVAC-SVC-MGMT `$`, IT-OPS-STARTER `$$`; basis in §3). No open authoring inputs remain; the only remaining gates are the per-step destructive sign-offs.

Three natures on `domains`: `established_market` (own masters + a `primary`-coverage vendor),
`emerging_market` (own masters, only `partial` coverage), `bundle` (no own masters, embeds/consumes only).
`industry_id` and segment/persona are orthogonal optional targets.

- [ ] Add `domains.domain_kind` as an enum field with `enum_values=['established_market','emerging_market','bundle']` (the platform enforces it from the field metadata per Rule #13 - do NOT hand-write a DB CHECK constraint), NOT NULL, default `established_market`.
- [ ] DEFER `domains.target_segment`: do NOT add it now - redundant with `min_org_size` for size, and the persona/use-case aspect has no defined vocab and no consumer (NEW-8). Revisit later with a proper enum + a real consumer if persona/use-case tagging is needed. [Q1: defer]
- [ ] `domains.industry_id` ALREADY EXISTS - do NOT add. Industry is a DOMAIN-grain concept: a module belongs to exactly one domain and inherits that domain's industry, so `domain_modules.industry_id` is redundant, not an independent axis. §3 sets industry on the bundle-domains; the module copy just inherits (ignore it).
- [ ] Do NOT add `catalog_release` to `domain_modules`; bundles publish via their promoted domain row (§5).
- [ ] RULE #8 BUNDLE CARVE-OUT (a bundle is NOT a market, so it carries the buyer-shape fields but NOT the market-research fields). On a `domain_kind='bundle'` row POPULATE: `min_org_size` (set), `cost_band` (the real "what does this lite product cost" signal), `certification_required` (false). Leave NULL and do NOT invent: `usa_market_size_usd_m` + `market_size_source_year` (a bundle has no TAM; fabricating one corrupts the exact substrate this taxonomy protects). `crud_percentage` / `business_logic` are derivable/optional, not load-bearing - leave at DB default / empty. §4 carves `domain_kind='bundle'` out of A1's market-field floor and makes the readers treat those fields as N/A (not zero) for bundles. NOTE (separate, not part of this migration): Rule #8 cites a `validateDomainRow()` hard-fail in `load_research.ts`, but no such function exists in any loader (only in SKILL.md / old plan docs). Nothing blocks bundle-domain creation at write time. Track the SKILL-vs-code drift as its own fix (implement the validator, or stop citing it as a guarantee).
